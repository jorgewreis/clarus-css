1) Visão de Produto e Meta de Mercado
Objetivo principal
Posicionar o Clarus CSS entre os frameworks CSS de maior excelência, com foco em:
• Produtividade real (utilitários + componentes prontos)
• Qualidade de engenharia (performance, acessibilidade, estabilidade)
• DX superior (documentação interativa, API previsível, migração simples)
• Escalabilidade (tokens, temas multi-brand, arquitetura modular)
Proposta de valor (posicionamento)
“Clarus CSS é um framework híbrido moderno: produtividade utilitária + consistência de componentes, com JS nativo opcional, acessibilidade robusta e arquitetura orientada a design systems.”

3) Princípios de Arquitetura
1. Progressive enhancement primeiro
 Componente funcional em HTML/CSS básico; JS apenas enriquece comportamento.
2. Acessibilidade por design
 Nenhum componente “completo” sem teclado + ARIA + foco + contraste.
3. Performance como requisito de produto
 Metas de tamanho e renderização em toda release.
4. API estável e previsível
 Sem quebrar uso sem justificativa; semver e política de depreciação claras.
5. Modularidade real
 Import por pacote/componente com tree-shaking.

5) Design Tokens e Theming (Nível Enterprise)
Estrutura de tokens
• Primitivos: --cl-color-blue-500, --cl-space-4, --cl-radius-md
• Semânticos: --cl-color-bg-surface, --cl-color-text-primary, --cl-color-border-default
• De componente: --cl-btn-bg, --cl-input-border-focus
Paleta algorítmica
Evoluir de hue fixo para estratégia híbrida:
• Base em OKLCH (melhor percepção visual)
• Fallback em HSL quando necessário
• Validação automatizada de contraste (AA/AAA)
Exemplo:
:root {
  --cl-primary-h: 250;
  --cl-primary-c: 0.16;
  --cl-primary-l-500: 0.62;
 
  --cl-primary-500: oklch(var(--cl-primary-l-500) var(--cl-primary-c) var(--cl-primary-h));
  --cl-primary-600: oklch(0.54 var(--cl-primary-c) var(--cl-primary-h));
}
Dark mode sem flash
• Separar FOUC de fallback sem JS
• FOUC: CSS crítico + tema inicial no <head> antes do primeiro paint
• Fallback sem JS: componentes essenciais com CSS puro (details/summary ou :checked)
Script de tema inicial:
<script>
(() => {
  const t = localStorage.getItem('theme');
  const m = window.matchMedia('(prefers-color-scheme: dark)').matches;
  document.documentElement.dataset.theme = t || (m ? 'dark' : 'light');
})();
</script>

15.2 Container queries
• Definir tokens de container (--cl-cq-sm, --cl-cq-md, --cl-cq-lg)
• Criar utilitários e exemplos oficiais
• Não substituir totalmente media queries globais
15.3 :has() com fallback
• Usar para melhoria contextual, não para lógica crítica
• Fornecer estilo base sem :has()
• Documentar comportamento por navegador
15.4 Theming multi-brand
• Tema base + temas por marca usando tokens semânticos
• Suporte por atributos:
• data-theme="light|dark"
• data-brand="x|y|z"
• Troca em runtime sem recompilar CSS
15.5 Purge e análise de CSS
• Pipeline de extração de classes usadas
• Safelist para classes dinâmicas
• Relatório de classes removidas por build
